Numbers

Basics

Concatenation

JS uses + as both addition and concatenation - adding strings together: 


    10 + 20; // 30
    "10" + "20"; // 1020 (a string)

Using any string will concatenate as a string: 


    10 + "20" // 1020 (a string)

Only the + operand affects concatenation: 


    var x = "100";
    var y = "10";
    var a = x / y;
    a // 10

    var b = x + y;
    b // 10010 (a string)

NaN - Not a Number

NaN - Not a Number indicates a number is not legal: var x = 100 / "Apple"; // NaN.

isNaN() is a global function used to find out if a variable is a number: var x = 100 / "Apple"; isNaN(x); // true.

Arithmetic

Operators

Unary is an operator with only one operand: -8; binary is an operator with two: y * x, or 0 + 8.

  • Addition + and Subtraction -
  • Multiplication * and Division /
  • Modulus %
  • Exponentiation **
    • Same result as Math.pow(x,y);, multiplies number x by itself y times.
    • Square root: x ** (1/2); Cubic root: x ** (1/3)
  • Increment ++ and Decrement --
    • Most commonly found in loops, cannot apply directly to a number but must be applied to a variable.
    • Can be a postfix counter++, or a prefix ++counter. If in a position to return to illustrate the difference, postfix returns the old number, then increments/decrements, while prefix will increment/decrement and then return the new number.

Operands/Arguments

The numbers in an operation, which can be literals: var x = 100 + 50;, variables: var x = a + b;, or expressions: var x = (100 + 50) * a;

Precedence

Determines the order expression is executed with more than one operator. Standard math precedence, though note unary (+, -) operators have higher precedence than binary (*, +, %, -, /) operators.

Assignment and modify-in-place operators have very low (level 3) precedence and usually performed last in the operation.

Assignment Operators

Assign a value to a variable and help keep code neater and more efficient; e.g. += addition assignment, *= multiplication assignment, etc.

  • x += 4; is the same as x = x + 4;
  • x /= 5; is the same as x = x / 5;, etc.

These help to modify in place - apply an operator to a variable, then store the new result in the same variable. Slow version: 


    let n = 2;
    n = n + 5;
    n = n * 2;
    n // 14

This can be condensed and more efficient: 


    let n = 2;
    n += 5;
    n *= 2;
    n // 14

Comparison Operators

These bad boys help with true/false tests, which can be used to display text labels if a feature is on/off (toggle), display game over messages, victory messages, correct seasonal greeting depending on season, or time, etc.

  • === strict equality tests if both value and type are identical
  • !== stric non-equality tests if they are not identical
  • < > <= >= are your classics
  • Note: != and == are valid, but they only test values, not value and datatypes so will result in more issues - use === !==

Decimals

To set a float to a specific number of decimals, use toFixed(): 


    let lotsOfDecimal = 1.1095809158;
    let twoDecimalPlaces = lotsOfDecimal.toFixed(2);
    twoDecimalPlaces // 1.11

Converting to Number Data Type

When a number is stored as a string, how can we convert it? Most common when form input as type 'text'. Pass the string value into Number() constructor: 


    let myNumber = '74';
    myNumber + 3; // 743 - still a string :(
    Number(myNumber) + 3; // 77

Unary + doesn't do anything to numbers, but if operand is not a number it will convert it into a number - performs the same function as Number(), but shorter: 


    let apples = "2";
    let oranges = "3";
    alert( apples + oranges ); // 23
    alert( +apples + +oranges ); // 5 - converted to numbers 
    alert( Number(apples) + Number(oranges)); // 5 - same thing, but longer

Unary + converts the strings to numbers first, then performs operation, due to having a higher precedence

Variables

Basics

Variables store goodies, data, visitors, etc. with let - let message = "hello".

When storing multiple variables, best to store on individual lines for readability: 


    let user = 'John';
    let age = 25;
    let message = 'Hello';

Or:


    let user = 'John',
    age = 25,
    message = 'Hello';

var is an older way to declare variables, but for now stick with let.

Note you can declare multiple variables and copy data from one to another, but cannot declare the same variable twice with different data: 


    let hello = "Hello world";
    let message;
    let message = hello;
    message // "Hello world"

    let message = 'this';
    let message = 'that'; // syntax error

Naming

Naming in JS has several limitations:

  • only letters, digits, or the symbols $ and _
  • the first character cannot be a digit
  • reserved names like let, class, return, etc. cannot be used

camelCase is used, case matters (e.g. apple and appLe are different), and should have a clean, obvious meaning:

  • Use human-readable names: userName, shoppingCart, newUser, etc.
  • Stay away from abbreviations or short names: a, mug, etc.

Constants

Unchanging variables can be declared with const rather than let: const myBirthday = '10.24'1985';. Trying to reassign a const returns an error

aliases can be indicated with uppercase constants, for difficult-to-remember values, using capitals and underscores: 


    const COLOR_RED = '#F00';
    const COLOR_GREEN = '#0F0';

    let color = COLOR_RED;

This can help make it easier to choose colors, prevent typos, or make document-wide changes.

Data Types

Basics

There are 8 main data types, which can be put into a variable (dynamically typed):

  1. Number - integer and floating point numbers, can use math operations
    • Special numeric values: infinity, -infinity, NaN
  2. BigInt - recently added to get over integer size issues for numbers
    • Created by adding nto end of an integer: bigInt = 123456789101112131564984n
  3. String - must be surrounded by quotes or backticks
  4. Boolean - true or false
  5. Null - represents nothing/empty/unknown
  6. Undefined - value is not assigned (variable is declared but not assigned
  7. Objects and symbols -
    • Object is special, in that it is the only non-primitive datatype (can contain more than 1 'thing' (string, number, etc.); used to store collections of data and complex entities
    • Symbol is used to create unique identifiers for objects
  8. Typeof operator returns the type of argument, good for a quick check. typeof x and typeof(x) are the same.

Strings

Basics

Use single or double quotes, or backticks, to surround a string; escape notation \ used for internal apostrophes/quotes: let bigMouth = 'I\'ve got no...';

Concatenating strings with + to join together; can repeat and mix variables and strings: 


    let one = 'hello, ';
    let multiple = one + one + one; // hello, hello, hello, 

    // mixing strings and variables:
    let response = one + 'I am fine - ' + question;

Template literals/strings is a newer way to provide easier-to-read strings, using backticks ` ` to wrap the entire string, and can include other variables/expressions with ${} (placeholder): 


    examReport = `You scored ${ examScore }/${ examHighestScore } (${ Math.round((examScore/examHighestScore*100))}. ${ examScore >= 49 ? ‘Well done, you passed!’ : ‘Bad luck, you didn\’t pass this time.’}`;

Newline character \n in traditional code is used to split a string over multiple lines: 


    output = 'I like the song "' + song + '".\nI give it a score of ...';
    //
    I like the song Rosetta Stoned.
    I give it a score of ...

Template literals respect line breaks, so if code is written with two lines, it will output with two lines.

String Methods and Properties

Basics

A method is a functionality built into a language or specific data types; these can be used on strings, such as indexOf(), search(), etc.

Note: all string methods return a new string - they do not modify the original string. They are immutable.

Misc:

  • .length to determine string length: var txt = "ababab"; var sln = txt.length; // 6
  • .indexOf() to find a text's starting index inside another string: 
    
    var str = 'banana eats';
    var pos = str.indexOf('eats'); // 7
  • .lastIndexOf() to find the last occurrence of a text - e.g. 'banana eats eats' would return the index of the second 'eats'
    • Both return -1 if the text is not found
    • Can specify the starting position/index (default 0) by inputting a second parameter: str.indexOf('eats', 6);
  • .search() to search for a string inside a string, which returns the position of the first occurrence of the specified text.
  • NOTE: search() cannot take a start-position parameter. indexOf() cannot take regular expressions (RegExp - for text search/replace operations) as inputs/search values.

Extracting String Parts:

3 methods to extract part of a string:

  • slice(start, end) extracts a part of a string and returns the extracted as a new string. If a parameter is negative, index is counted from end of string. If the second parameter is omitted, slices out the remainder of the string: 
    
    let str = "Apple, Banana, Kiwi";
    let res = str.slice(7, 13); // Banana 

    str.slice(-12, -6); // Banana 

    str.slice(7) // Banana
  • substring() is similar to slice(), but cannot accept negative indices. If second parameter omitted, slices out the remainder of the string.
  • substr() similar to slice(), but the second parameter specifies the length of what is extracted (e.g. str.substr(7, 6) starts at index 7 and slices next 6 indices). If second parameter is omitted, slices out rest of string; if first parameter is negative, counts from end of the string.

Replacing String Content:

  • replace() replaces (ha) a specified value with another value in a string: 
    
    str = "Please eat at Burt\'s";
    let clocks = str.replace('Burt\'s', 'Clocks');
    clocks; // "Please eat at Clocks"
  • Does not change the string initially called on, but returns a new string. By default, replaces only the first match if duplicate text in string, and case sensitive!
    • An insensitive RegExp can replace case sensitivity with /i tag: str.replace(/BURTS/i, 'Clocks');
    • A global RegExp can replace ALL matches with /g tag: str.replace(/Burts/g, 'Clocks').

Converting to Upper/Lower case:

  • toUpperCase() and toLowerCase(), aptly named: 
    
    let text1 = "Hello World!";
    let text2 = text1.toUpperCase(); // HELLO WORLD!

    let text3 = text1.toLowerCase(); // hello world!

Extracing String Characters:

3 methods to extract:

    • let str = "HELLO WORLD!"; str.charAt(0); // H

String Conditionals

Comparisons: < > == != return a boolean result. Can even assign result as a variable: let result = 5 > 4; alert( result ); // true

String Comparison - JS uses 'dictionary' or 'lexicographical' - e.g. strings compared letter by letter using the following algorithm:

  1. Compare the first character of both strings.
  2. If 1st character from 1st string is greater than the other string's, the 1st string is greater and we're done. (Or vice-versa).
  3. Otherwise, if both 1st characters are the same, compare the 2nd characters the same way.
  4. Repeat until end of either string.
  5. If both end with the same length, then they are equal; otherwise the longer string is greater.
  6. Note: it's unicode order, which is close to alphabetical - but A is not equal to a!

Conditionals

Ever want to perform different actions depending on different situations? Well, here you go!

Conditional Statements

if

if to specify a block of code to be executed IF a specified condition is true.

Evaluates a condition in parentheses, and if true, executes a block of code. Note 0 '' null undefined NaN all become falsy values.

Syntax: 


    if (condition) {
        // block of code to execute if condition is true
    }

    // example:
    if (hour < 18) {
        greeting = "Good day";
    }

  • Can be combined with question mark/ternary operator ? for a simple true/false check: 
    
    let result = condition ? value1 : value2;

    // example:
    let greeting = ( is Birthday ) ? 'happy birthday' : 'good morning';

    // theme chooser:
    const select = document.querySelector('select');
    const html = document.querySelector('html');

    function update(bgColor, textColor) {
        html.style.backgroundColor = bgColor;
        html.style.color = textColor;
    }

    select.onchange = function() {
        ( select.value === 'black' ) ? update('black', 'white') : update('white', 'black');
    }

    // example:

    let machineActive = true;
    let pwd = 'cheese';

    let machineResult;
    let pwdResult;

    if(machineActive === true) {
        ( pwd === 'cheese' ) ? pwdResult = "Welcome!" : pwdResult = "No luck!";
    } else {
        machineResult = "Sorry, the machine is not on";
    }

    // example:

    ( pwd === 'cheese' ) ? pwdResult = "Welcome!" : pwdResult = "No luck!";

    // multiple ternary example:

    let age = prompt('age?', 18);

    let message = (age < 3) ? 'Hi, baby!' :
        (age < 18) ? 'Hello!' :
        (age < 100) ? 'Greetings!' :

    // say 'Hello!', if false, then if age < 100, say 'Greetings!', if false, 
    // then say 'What an unusual age!'
  • If the condition is truthy, executes value1, otherwise executes value2.
  • Can sometimes be used to replace if, but not recommended as it's less readable. Same as if else.

else

else to specify a block of code to be executed if same condition is false.

Syntax: 


    if (condition) {
        // block of code
    } else {
        // block of code 
    }                    
    
    // example:
    if (hour < 18) {
        greeting = "Good day";
    } else {
        greeting = "Good evening";
    }

else if to specify a new condition to test if the first is false.

Syntax: 


    if (condition1) {
        // block of code if condition1 = true 
    } else if (condition2) {
        // block of code if condition1 is false and condition2 is true 
    } else {
        // block of code if condition1 and condition2 are false 
    }

    // example:
    if (time < 10) {
        greeting = "Good morning";
    } else if (time < 20) {
        greeting = "Good day";
    } else {

    // example several options:
    if (choice === 'sunny') {

switch

switch is an alternative to if else when there's too much code or the conditions are too complex - or where you just want to set the variable to a certain choice/value.

switch statements take a single expression/value as input and look through a number of choices (switch ranges) until a match is found and corresponding code is executed.

Syntax: 


    switch (expression) {
        case choice1:
            // run this code
            break;    
        case choice2:
            // run this code instead
            break;    
        // repeat as needed   
        default:
            // actually, just run this code
    }
    
    // example:
    
    function setWeather() {
        const choice = select.value;
    
        switch (choice) {
            case 'sunny':
                para.textContent = 'Wear Shorts!';
                break;
            case 'rainy':
                para.textContent = 'Wear a raincoat!';
                break;
            case 'snowing':
                para.textContent = 'Wear gloves!';
                break;
            default:
                para.textContent = '';
        }
    }

    // example:

    let response;
    let score = 75;
    let machineActive = false;

    if(machineActive) {
        
        switch (true) {
            case (score < 00 || score > 100):
                response = "This is not possible, an error has occurred";
                break;
            case (score >= 0 && score < 20):
                response = "That was a terrible score - total fail!";
                break;
            case (score >= 20 && score < 40):
                response = "You know some things, but it's a pretty bad score. Needs improvement";
                break;
            case (score >= 40 && score < 70):
                response = "You did a passable job, not bad!";
                break;
            case (score >= 70 && score < 90):
                response = "That\'s a great score, you really know your stuff";
                break;
            case (score >= 90 && score < 101):
                response = "What an amazing score! Did you cheat? Are you for real?"
        }

    } else {
        response = 'The machine is turned off. Turn it on to process your score.';
    }

Logical operators

There are 3 global operators in JS:

|| OR - if any arguments are true, it returns true, otherwise returns false: 


    alert( true || true); // true 
    alert( false || true); // true 
    alert( true || false ); // true 
    alert( false || false ); // false 

    example:
    if (iceCreamVanOutside || houseStatus === 'on fire') {
        console.log('You should leave the house quickly');
    } else {
        console.log('Probably should just stay in then');
    }

If operand is not boolean, it is converted for evaluation; 1 = true, 2 = false; ( 1 || 0 ) is the same as ( true || false ).

Mostly used with an if statement to test of any are true: 


    let hour = 9;
    if (hour < 10 || hour > 18 ) {
        alert( 'The office is closed.' );
    }

Can have multiple conditions - e.g. before 10, after 18, or weekend: 


    let hour = 12;
    let isWeekend = true;
    if (hour < 10 || hour > 18 || isWeekend) {
        alert( 'The office is closed.' );
    }

OR finds the first truthy value:

  1. Evaluates from left to right.
  2. For each operand, converts to boolean; if true, stops and returns the original value of operand.
  3. If all operands have been evaluated (ie all are false), returns the last operand.
  4. If all false, it returns the last value: 
    5. 
    let firstName = "";
    let lastName = "";
    let nickName = "";
    alert( firstName || lastName || nickName || "Anonymous" ); // Anonymous

Short-circuit evaluation - once it sees true, it stops and returns that value and doesn't even look at other values: 


    true || alert("not printed");
    false || alert("printed");
        // in the first line, OR saw true on the left so returns without even looking at alert("not printed");
        // in the second line, OR saw false, so looked at it and returned the second line

Also need to make sure not to give a list to return true, must be specific: 


    if (x === 5 || 7 || 10 || 20) {
        // bad bad bad, won't work 
    }

    if (x === 5 || x === 7 || x === 10 || x === 20) {
        // will compare each of these and execute code accordingly
    }

&& AND - both operands have to be true, or returns false: 


    alert( true && true ); // true 
    alert( false && true ); // false 
    alert( true && false ); // false 
    alert( false && false ); // false

    let hour = 12;
    let minute = 30;
    if (hour == 12 && minutes == 30) {
        alert( 'The time is 12:30' );
    }

    // example:
    if (choice === 'sunny' && temperature < 86) {
        para.textContent = 'It is ' + temperature + ' degrees outside - nice and sunny';
    } else if (choice === 'sunny' && temperature >= 86) {
        para.textContent = 'It is ' + temperature + ' degrees outside - way too hot!';
    }

AND finds the first falsy value:

  1. Evaluates from left to right.
  2. Each operand converts to boolean - if result is false, stops and returns original value of that operand.
  3. If all have been evaluated (and all truthy), returns the last operand - e.g. returns the first falsy, or the last value if no false is found
  4. NOTE: precedence of AND is higher than OR: a && b || c && d -> the two && comparisons are done first, then ||.

! NOT - result = !value

  1. Converts the operand to boolean true/false.
  2. Returns the inverse value: 
    
    alert( !true ); // false
    alert( !false ); // true 

    example:
    if (!(iceCreamVanOutside || houseStatus === 'on fire')) {
        console.log('Probably should just stay in then.');
    } else {
        console.log('You should leave the house quickly.');
    }

    3. Highest precedence of all logical operators - execute first.

Functions

Basics

Functions allow you to store a piece of code that does a single task, but call that code in a short command; Some functions are built-in to the browser and not part of the JS language.

Functions

vs

Methods

  • A method is a function that is part of an object - more later.
  • custom functions are defined in the code, not built-in to the browser.
  • functions can call inside other functions

Naming

Functions are actions, so name around verbs and provide a brief, accurate description:

showMessage(..), getAge(..), calcSum(..), etc.

Functions should do what the name suggests! Two independent actions? Use two functions.

Invoking

Just run it using it's name().

Return values

Not all functions return a value; they are void, or undefined; generally used when the function is an intermediate step in a calculation.

To return a value, return can be any place in the function; when it is reached, the function stops and the value is returned. If empty or without a return, it is undefined.

Function types

Function declaration

The standard function syntax, declared as a separate statement in main code flow. Can be called earlier than defined - JS prepares the script and looks for/creates global functions in an initialization stage. If located within a code block, it's visible everywher inside, but not outside. 


    // syntax:
    function sayHi() {
        // code
    }

Function expression

Another syntax to create a function; created when execution reaches it and is usable only in that moment. 


    // syntax:
    let sayHi = function() {
        // code 
    }

Choosing between declaration and expression:

Declaration syntax allows more freedom in organizing the code, because they can be called before they're declared in the script, and is also better for readability.

Anonymous functions

A function with no name! Something something Jon Snow, I don't know, I never saw the show. Usually just use function() { // code }, and most often used with an event handler.

Callback functions

Declaring a function inside a call, building parameters as functions. Pass a function and expect it called back later if necessary - see example html file 3. These functions are only inside the call - hyper local!

Arrow functions

Often better than expressions; very convenient for simple one-line actions to not write so many words.

  • let func = (arg1, arg2, ...argN) => expression
  • If only one argument, can omit parens: func = arg => expression.
  • If no argument, empty parens: func = () => expression.
  • multiline arrow functions allow added complexity: 
    
    // example:

Parameters

Parameters, aka arguments, properties, and attributes, are values included in a function's () to do its job properly - they are what feed the function.

If a parameter is not included, it becomes undefined and not returned.

A default value can be assigned: 


    let from = Ann;
    function showMessage(from, text = "no text given") // Ann, no text given
Can also use alternative default values in the body of the code rather than the declaration.

Scope and conflicts

Scope is when values and expressions are 'visible' and able to be referenced. They can be layered in the hierarchy so the child(ren) has access to parent scopes, but not the other way around.

Functions are 'closers' in JS, in that they create a scope in which something defined within the function cannot be accessed outside or in other functions.

Global scope is the top level outside all functions, and can be accessed within all functions.

Local variables are part of the scope - declared in a function, and only seen inside that function.

Outer variables, aka global variables, are declared outside a function, can modify an outer value, and can use the same name within a function without affecting the outer variable. Try to use global variables as infrequently as possible - mostly used to store project-level data!

Arrays

Basics

An array is a special variable which can hold more than one value at once; rather than storing car1, car2, etc., can use array cars. Items can be accessed using an index number - cars[1].

Creating, accessing, changing:

To create an array: 


    // syntax:
    var arrayName = [item1, item2, item3, ...];
    // can span multiple lines:
    var cars = [
        "Saab",
        "Volvo",
        "BMW"
        ];

Note: avoid using new to create an array, if examples are seen; new can complicate the code and cause issues (e.g. potentially creating an array with undefined elements)! 


        var points = new Array(40, 100); // creates array w/2 elements, 40 and 100
        var points = new Array(40); // creates array with 40 undefined elements!

Use index numbers to access the elements of an array - reminder they start at 0! e.g. var name = cars[0]; // Saab

To access the full array, simply is the name: document.getElementById('demo').innerHTML = cars;

To change an array element, simply redefine using the index: cars[0] = "Saab"; cars[0] = "Opel";

Object?

Yeppers.

But they're a special kind of object. This allows variables of different types within the same array - e.g. objects, functions, arrays, etc: 


    myArray[0] = Date.now;
    myArray[1] = myFunction;
    myArray[2] = myCars;

Difference between an array and object? Arrays use numbered indices, whereas objects use named indices.

Use an object when you want element names to be strings; use an array when you want element names to be numbers.

Using typeof on an array returns object - but this can cause issues. Instead, use Array.isArray() (e.g. Array.isArray(fruits); // returns true), or use instanceof, which returns true with a given constructor: fruits instanceof Array; // returns true.

Array properties and methods

  • var x = cars.length; // returns the number of elements; reminder will by 1 more than highest array index because it starts at 0!

Best Practices

Understand the problem

The first step of any process; write it down, reword it into plain English until it makes sense - use diagrams if needed!

Plan! Ask questions - does it have a ui? what will it look like/functionality will it have? What inputs will it have? Will the user enter data, or is the data from elsewhere? What's the output? What are the steps necessary to get the output from the inputs?

Pseudo Code

The algorithm - the recipe to solve the problem - laid out in logical steps in plain language, not code. 


    // user inputs number
    // initialize a counter variable and set value to 0
    // while counter is smaller than input number, increment counter by 1
    // print value of the counter variable

While planning, identify smaller problems of the bigger problem - these are the steps of the algorithm; pick small(er) ones and start coding! Reminder, the algorithm may be incomplete as not all steps are known or thought of right away.

To reiterate the process:

  • Understand - know what's being asked until it can be explained in plain language - write it down, draw diagrams, etc. if needed to understand
  • Plan - write down the steps needed - answer those questions above
  • Divide - break into small problems, and work to solve one by one with the simplest
  • Stuck? Debug - go step by step to find where went wrong; reassess - take a step back, look from a different perspective; research - Google it!

Clean Code

  • Indentation - be consistent!
  • Semicolons - mostly optional in JS, but just get used to it for clarity
  • Line length - limit the length of each line, manually breaking longer than ~80 characters
  • Naming - be descriptive, and use camelCase. Variables with nouns, functions with verbs. It's OK to use single-character variables when using a loop or callback function.
