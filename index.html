Numbers

Basics

Concatenation

JS uses + as both addition and concatenation - adding strings together: 


    10 + 20; // 30
    "10" + "20"; // 1020 (a string)

Using any string will concatenate as a string: 


    10 + "20" // 1020 (a string)

Only the + operand affects concatenation: 


    var x = "100";
    var y = "10";
    var a = x / y;
    a // 10

    var b = x + y;
    b // 10010 (a string)

NaN - Not a Number

NaN - Not a Number indicates a number is not legal: var x = 100 / "Apple"; // NaN.

isNaN() is a global function used to find out if a variable is a number: var x = 100 / "Apple"; isNaN(x); // true.

Arithmetic

Operators

Unary is an operator with only one operand: -8; binary is an operator with two: y * x, or 0 + 8.

  • Addition + and Subtraction -
  • Multiplication * and Division /
  • Modulus %
  • Exponentiation **
    • Same result as Math.pow(x,y);, multiplies number x by itself y times.
    • Square root: x ** (1/2); Cubic root: x ** (1/3)
  • Increment ++ and Decrement --
    • Most commonly found in loops, cannot apply directly to a number but must be applied to a variable.
    • Can be a postfix counter++, or a prefix ++counter. If in a position to return to illustrate the difference, postfix returns the old number, then increments/decrements, while prefix will increment/decrement and then return the new number.

Operands/Arguments

The numbers in an operation, which can be literals: var x = 100 + 50;, variables: var x = a + b;, or expressions: var x = (100 + 50) * a;

Precedence

Determines the order expression is executed with more than one operator. Standard math precedence, though note unary (+, -) operators have higher precedence than binary (*, +, %, -, /) operators.

Assignment and modify-in-place operators have very low (level 3) precedence and usually performed last in the operation.

Assignment Operators

Assign a value to a variable and help keep code neater and more efficient; e.g. += addition assignment, *= multiplication assignment, etc.

  • x += 4; is the same as x = x + 4;
  • x /= 5; is the same as x = x / 5;, etc.

These help to modify in place - apply an operator to a variable, then store the new result in the same variable. Slow version: 


    let n = 2;
    n = n + 5;
    n = n * 2;
    n // 14

This can be condensed and more efficient: 


    let n = 2;
    n += 5;
    n *= 2;
    n // 14

Comparison Operators

These bad boys help with true/false tests, which can be used to display text labels if a feature is on/off (toggle), display game over messages, victory messages, correct seasonal greeting depending on season, or time, etc.

  • === strict equality tests if both value and type are identical
  • !== stric non-equality tests if they are not identical
  • < > <= >= are your classics
  • Note: != and == are valid, but they only test values, not value and datatypes so will result in more issues - use === !==

Decimals

To set a float to a specific number of decimals, use toFixed(): 


    let lotsOfDecimal = 1.1095809158;
    let twoDecimalPlaces = lotsOfDecimal.toFixed(2);
    twoDecimalPlaces // 1.11

Converting to Number Data Type

When a number is stored as a string, how can we convert it? Most common when form input as type 'text'. Pass the string value into Number() constructor: 


    let myNumber = '74';
    myNumber + 3; // 743 - still a string :(
    Number(myNumber) + 3; // 77

Unary + doesn't do anything to numbers, but if operand is not a number it will convert it into a number - performs the same function as Number(), but shorter: 


    let apples = "2";
    let oranges = "3";
    alert( apples + oranges ); // 23
    alert( +apples + +oranges ); // 5 - converted to numbers 
    alert( Number(apples) + Number(oranges)); // 5 - same thing, but longer

Unary + converts the strings to numbers first, then performs operation, due to having a higher precedence

Variables

Basics

Variables store goodies, data, visitors, etc. with let - let message = "hello".

When storing multiple variables, best to store on individual lines for readability: 


    let user = 'John';
    let age = 25;
    let message = 'Hello';

Or:


    let user = 'John',
    age = 25,
    message = 'Hello';

var is an older way to declare variables, but for now stick with let.

Note you can declare multiple variables and copy data from one to another, but cannot declare the same variable twice with different data: 


    let hello = "Hello world";
    let message;
    let message = hello;
    message // "Hello world"

    let message = 'this';
    let message = 'that'; // syntax error

Naming

Naming in JS has several limitations:

  • only letters, digits, or the symbols $ and _
  • the first character cannot be a digit
  • reserved names like let, class, return, etc. cannot be used

camelCase is used, case matters (e.g. apple and appLe are different), and should have a clean, obvious meaning:

  • Use human-readable names: userName, shoppingCart, newUser, etc.
  • Stay away from abbreviations or short names: a, mug, etc.

Constants

Unchanging variables can be declared with const rather than let: const myBirthday = '10.24'1985';. Trying to reassign a const returns an error

aliases can be indicated with uppercase constants, for difficult-to-remember values, using capitals and underscores: 


    const COLOR_RED = '#F00';
    const COLOR_GREEN = '#0F0';

    let color = COLOR_RED;

This can help make it easier to choose colors, prevent typos, or make document-wide changes.

String Comparison - JS uses 'dictionary' or 'lexicographical' - e.g. strings compared letter by letter using the following algorithm:

  1. Compare the first character of both strings.
  2. If 1st character from 1st string is greater than the other string's, the 1st string is greater and we're done. (Or vice-versa).
  3. Otherwise, if both 1st characters are the same, compare the 2nd characters the same way.
  4. Repeat until end of either string.
  5. If both end with the same length, then they are equal; otherwise the longer string is greater.
  6. Note: it's unicode order, which is close to alphabetical - but A is not equal to a!

Functions

Basics

Functions allow you to store a piece of code that does a single task, but call that code in a short command; Some functions are built-in to the browser and not part of the JS language.

Functions vs Methods

  • A method is a function that is part of an object - more later.
  • custom functions are defined in the code, not built-in to the browser.
  • functions can call inside other functions

Naming

Functions are actions, so name around verbs and provide a brief, accurate description:

showMessage(..), getAge(..), calcSum(..), etc.

Functions should do what the name suggests! Two independent actions? Use two functions.

Invoking

Just run it using it's name().

Not all functions return a value; they are void, or undefined; generally used when the function is an intermediate step in a calculation.

To return a value, return can be any place in the function; when it is reached, the function stops and the value is returned. If empty or without a return, it is undefined.

  • If no argument, empty parens: func = () => expression; - e.g. let sayHi = () => alert("Hello!");
  • multiline arrow functions allow added complexity: 
    
    /* example: */

    let sum = (a, b) => { // the curly brace opens a multiline function
        let result = a + b;
        return result; // if we use curly braces, then we need an explicit 'return'
    };
    
    alert( sum(1,2) ); // 3

Parameters

Parameters, aka arguments, properties, and attributes, are values included in a function's () to do its job properly - they are what feed the function. 


    function showMessage(from, text) { // arguments: from, text
        alert(from + ': ' + text);
    }
    
    showMessage('Ann', 'Hello!'); // Ann: Hello!
    showMessage('Nick', 'Turtle'); // Nick: Turtle

If a parameter is not included, it becomes undefined and not returned.

A default value can be assigned: 


    let from = Ann;
    function showMessage(from, text = "no text given") // Ann, no text given 
                        
 
    /* Can also be a more complex expression, only evaluated and assigned if param missing: */

    function showMessage(from, text = anotherFunction()) {
        // anotherFunction() is only executed if no text is given
        // its result becomes the value of text
    }
Can also use alternative default values in the body of the code rather than the declaration.

Scope and conflicts

Scope is when values and expressions are 'visible' and able to be referenced. They can be layered in the hierarchy so the child(ren) has access to parent scopes, but not the other way around.

Functions are 'closers' in JS, in that they create a scope in which something defined within the function cannot be accessed outside or in other functions.

Global scope is the top level outside all functions, and can be accessed within all functions.

Local variables are part of the scope - declared in a function, and only seen inside that function.

Outer variables, aka global variables, are declared outside a function, can modify an outer value, and can use the same name within a function without affecting the outer variable. Try to use global variables as infrequently as possible - mostly used to store project-level data!

        console.log("Inside function");
        console.log(x);
    }
    
    console.log(x); // causes error

        console.log("inside function");
        console.log(x);

    console.log("Outside function");
    console.log(x);

Arrays

Basics

An array is a special variable which can hold more than one value at once; rather than storing car1, car2, etc., can use array cars. Items can be accessed using an index number - cars[1].

Creating, accessing, changing:

Loops

Basics

A loop is useful to do the same thing over and over rapidly, repeating repetitive tasks, or iterating.

Features of a loop:

  • Counter/initializer - initialized with a certain value; is the starting point, usually set to a number, and incremented/decremented to count the number of times the loop has run
  • Condition - true/false to determine if the loop continues or stops, generally features a comparison operator.
  • Iterator/final-expression - generally increments (but can decrement) the counter by a small amount until the condition is no longer true; always run each time the loop has gone through a full iteration. 

    // counter, condition, iterator; in pseudocode:
    loop(food = 0; foodNeeded = 10) {
        if (food >= foodNeeded) {
            exit loop;
            // we have enough food, let's go home
        } else {
            food += 2; // spend an hour collecting 2 more food
            // loop will then run again
        }
    }

Which should you use? Whichever you prefer, they're largely interchangeable; most sources recommend the for loop, as it's easiest to remember and neater.

For loop

For loops consist of an initializer, condition, and final-expression, which is always evaluated each time the loop goes through a full iteration - usually increment or decrement (e.g. i++). 


    // syntax:
    for (initializer; condition; final-expression) {
        // code to run
    }

    // example:
    const cats = ['Bill', 'Jeff', 'Pete', 'Biggles', 'Jasmin'];
    let info = 'My cats are called ';
    const para = document.querySelector('p');

    for (let i = 0; i < cats.length; i++) {
        info += cats[i] + ', ';
    }

    para.textContent = info;
        // returns My cats are called Bill, Jeff, Pete, Biggles, Jasmin, 

    // note it ended in a comma + space? The fix:
    for (let i = 0; i < cats.length; i++) {
        if (i === cats.length - 1) {
            info += 'and ' + cats[i] + '.';
        } else {
            info += cats[i] + ', ';
        }
    }

Exiting loops with break

Use a break to exit a loop before all iterations are completed; will immediately exit the loop and move on to the next code. For when the condition must be checked in the middle of the loop, not the beginning or end. 


    let sum = 0;
    while (true) {
        let value = +prompt("Enter a number", '');
        if (!value) break; // if it's not a value, exit the loop
        sum += value;
    }
    alert( 'Sum: ' + sum );

For example, if you want to find contacts/phone numbers, loop until the number is found, then break to stop the function from iterating further. 


    // HTML: 
    <label for="search">Search by contact name: </label>
    <input id="search" type="text">
    <button>Search</button>

    const contact = ['Chris:2232322', 'Sarah:3453456', 'Bill:7654322', 'Mary:9998769', 'Dianne:9384975'];

    const input = document.querySelector('input');
    const btn = document.querySelector('button');

    // event listener to btn ('button') to perform the code when pressed
    btn.addEventListen('click', function() {
        // search value from text input stored in searchName, toLowerCase for case-sensitivity
        let searchName = input.value.toLowerCase();
        input.value = '';
        input.focus();
        for (let i = 0; i < contacts.length; i++) {
            // split current contact at the colon, storying in a new array called splitContact (in spots 1 and 2)
            let splitContact = contacts[i].split(':');
            // determine if searchName and the first part of the new array match:
            if (splitContact[0].toLowerCase() === searchName) {
                // if a match, combine with splitContact 1 (phone number):
                para.textContent = splitContact[0] + '\'s number is ' + splitContact[1] + '.';
                break;
            } else {
                // if no match, say so:
                para.textContent = 'Contact not found.';
            }
        }
    });

Skipping iterations with continue

Similar concept as break, but rather than breaking out of the loop, it will skip to the next iteration. 


    // to see if it's odd
    for (let i = 0; i < 10; i++) {
        // if true, skip the remaining part of the body
        if (i % 2 == 0) continue;
        alert(i); // 1, 3, 5, 7, 9
    }

Does not stop the whole loop, just the current iteration - and starts a new one (if the condition allows).

Example would be searching for square roots - if !== sqRoot, continue will start back at the counter. 


    // example looking for square roots or something:
    let num = input.value;

    for (let i = 1; i <= num; i++) {
        // find square root of each number w/Math.sqrt(i), then check if an integer by
        // testing if same as self when rounded down to nearest integer (Math.floor())
        let sqRoot = Math.sqrt(i);
        // if square root and rounded down do not equal one another, it means the square 
        // root is not an integer, so we're not interested in it - so continue to the 
        // next iteration of the loop
        if (Math.floor(sqRoot) !== sqRoot) {
            continue;
        }
        // if it is a square loop, skip the continue section and concatenate into paragraph:
        para.textContent += i + ' ';
    }

Labels for break/continue

To break out of multiple loops at once, use labels, an identifier, to specify: 


    labelName:
    for (...) {
        // code 
    }

    break labelName; // breaks to the outer loop 
                        

    // example:
    outer: for (let i = 0; i < 3; i++) {
        for (let j = 0; j < 3; j++) {
            let input = prompt(`Value at coords (${i},${j})`, '');
    
            // if an empty string, or canceled, then break out of both loops:
            if (!input) break outer; // break to 'outer' loop, since that's label name
    
            // do something with the value
        }
    }
    alert('Done!');
    
    // label can also be on a separate line:
    outer:
    for (let i = 0; ...)

Note labels don't allow jumping anywhere - can only jump above in a nested loop.

While

Similar to the for loop, except the initializer is set before the loop and final-expression is inside the loop after the code is run. When the condition is truthy, the code is executed.

Curly braces aren't required for a single-line body: while (I) alert (i--);

Objects

Basics

As a reminder, there are 8 data types - 7 are 'primitive' in that they contain a single thing, but objects are the exception.

Objects are used to store keyed collections of various data/complex entities, related data/functionality, usually of several variables and functions called properties and methods.

Can be created with {} and properties - in a key: value pair. Object literal syntax is most common and is to create an object as it's defined - e.g. const person = {};

Square brackets are more complex and powerful than dot notation, but more cumbersome and not as easy to use; use dot notation where property names are known and simple, else use square brackets.

To test if a property exists, use in - no error if property doesn't exist, just 'undefined'. alert( "age" in user ); // true, user.age exists.

How are things ordered inside an object? If we loop over one, we see integer properties are in numeric sort, and the rest in creation order.

  • Note that integer properties can be converted to string/integer without change: e.g. "49" can turn into 49; but "+49" can 'cheat' the system, as it can't be converted to an integer!

Speaking of looping, the for...in loop is a special form that walks over all keys of an object, and can use any variable name in the syntax.

Bracket Notation

Using square brackets to add/remove/get multiword properties:

  • user["likes birds"] = true;, etc.
  • Calling info: let age = 30;, user[age]; // 30
  • Similar to an array (name[1]) - call using the member name: person['age'];, person['name'], etc.
  • To set object members, Bracket notation is nice because you can create new member names, too: // declare vars; person[myDataName] = myDataValue;

Dot Notation

The namespace is the object name (e.g. person), followed by ., then the item to access. E.g. person.age;, person.interests[1];, person.bio();, etc.

Sub-namespaces are an object within an object (e.g. name: { first: 'bob', last: 'smith' }). Can be called with person.name.last, person.name.first, etc.

To set object members, simply person.age = 45;. person.age; // 45, etc.

'this'?

We'll get more to this later, but ensures the code refers to the current object, in case value's context changes.

Best Practices

Understand the problem

The first step of any process; write it down, reword it into plain English until it makes sense - use diagrams if needed!

Plan! Ask questions - does it have a ui? what will it look like/functionality will it have? What inputs will it have? Will the user enter data, or is the data from elsewhere? What's the output? What are the steps necessary to get the output from the inputs?

Pseudo Code

The algorithm - the recipe to solve the problem - laid out in logical steps in plain language, not code. 


    // user inputs number
    // initialize a counter variable and set value to 0
    // while counter is smaller than input number, increment counter by 1
    // print value of the counter variable

While planning, identify smaller problems of the bigger problem - these are the steps of the algorithm; pick small(er) ones and start coding! Reminder, the algorithm may be incomplete as not all steps are known or thought of right away.

To reiterate the process:

  • Understand - know what's being asked until it can be explained in plain language - write it down, draw diagrams, etc. if needed to understand
  • Plan - write down the steps needed - answer those questions above
  • Divide - break into small problems, and work to solve one by one with the simplest
  • Stuck? Debug - go step by step to find where went wrong; reassess - take a step back, look from a different perspective; research - Google it!

Clean Code

  • Indentation - be consistent!
  • Semicolons - mostly optional in JS, but just get used to it for clarity
  • Line length - limit the length of each line, manually breaking longer than ~80 characters
  • Naming - be descriptive, and use camelCase. Variables with nouns, functions with verbs. It's OK to use single-character variables when using a loop or callback function.
